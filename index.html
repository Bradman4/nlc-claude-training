Important Things to Know Upfront

⚠️Not a search engine: Claude's knowledge has a cutoff date. For the very latest NEC code updates or OSHA rule changes, always verify with official sources: OSHA 1910.269 ↗ · NEC (NFPA 70) ↗ · NESC (IEEE) ↗
⚠️Can make mistakes: Always double-check safety-critical, regulatory, or technical specifications before acting on them. When in doubt, verify directly with OSHA.gov ↗ or your compliance team.
✅No memory by default: Each new conversation starts fresh. Claude does not remember previous chats unless you opt in to the memory feature in Settings. Once enabled, Claude can recall past conversations — but here's what to know:

🔧 How to Turn Memory On:
1
Click your profile icon in the bottom-left corner of Claude.ai
2
Select Settings from the menu
3
Click Capabilities in the left menu
4
Toggle on "Search and reference past chats" and "Generate memory from chat history"
• Memory is project-scoped — each Project keeps its own separate memory.

• Does it max out? Claude operates within a 200,000 token context window. As memory grows, older content may be trimmed automatically.

• You stay in control — view, edit, or delete memory anytime via Settings → Capabilities. Choose Pause (stops adding new) or Reset (permanently deletes all).

• Best practice: Keep project instructions concise and use Projects — they only load relevant content into memory.

• For everyday NLC tasks — writing briefings, quizzes, and emails — you are very unlikely to ever hit a memory limit.

📁 What Are Projects?

Projects are dedicated workspaces that let you organize conversations around a specific topic. Claude remembers context across multiple conversations so you never have to re-explain your situation.

💡Think of a Project like a folder — everything related to a curriculum unit, safety program, or recurring report lives in one place.
📌Projects retain context: Claude remembers your instructions, preferences, and past conversations within the Project.
🗂️Great for ongoing work: Curriculum development, safety documentation, student communications, and recurring tasks.

Writing Great Prompts

The #1 skill to get the most out of Claude

Anatomy of a Great Prompt

The best prompts have up to 4 elements. More context = better results.

You are a technical training developer. I work at Northwest Lineman College and I'm building a new module on underground cable fault locating. Write a learning objective for this module aimed at apprentice linemen. Use the format: "After completing this module, students will be able to..." Keep it to one sentence.
Role (optional)
Context / Background
Task (what you want)
Format / Constraints

Example Prompts — Click to Copy

Summarize the following email and list any action items I need to take: [paste email here]
Act as a technical editor. Make these instructions clearer for a first-year apprentice: [paste text here]
Explain the difference between series and parallel circuits like I've never worked in electricity before. Use a simple analogy.
Write a professional but friendly follow-up email to a student who missed their scheduled lab session.
I need to choose between two approaches to teaching this topic. Here's my situation: [describe]. Give me pros and cons for each option.

Do's and Don'ts

✅ Do This

  • Be specific about what you want
  • Give background/context
  • Specify the format (email, bullet list, outline)
  • Mention your audience
  • Follow up to refine the answer

❌ Avoid This

  • Vague one-word prompts ("help")
  • Assuming Claude knows your role
  • Asking multiple unrelated things at once
  • Giving up after one bad response

🔤 What Is a Token?

Tokens are the units Claude uses to process text — both what you send and what it responds with. Think of tokens as roughly ¾ of a word.

📄
~750 words
= 1,000 tokens
📃
~75,000 words
= 100,000 tokens (~300 pages)
📚
~150,000 words
= 200,000 tokens (standard limit)

Every message you send AND every response Claude gives uses tokens from your context window. Long conversations or large uploaded files consume tokens faster.

📋 When Should You Request Enterprise?

Contact your IT administrator or Quanta AI program owner to request an Enterprise license upgrade if you experience any of the following:

🔴Hitting usage limits regularly — if you're getting "usage limit reached" messages multiple times a week
🔴Processing very large documents — multiple lengthy files that exceed the 200K token window in a single session
🔴Team-wide heavy usage — multiple people on your team hitting limits simultaneously
🔴Compliance requirements — your work involves HIPAA-regulated data, audit logs, or SSO requirements
🔴Departmental or org-wide deployment — rolling Claude out to an entire team or business unit
⚡ How to Maximize Your Token Usage — So You Don't Need an Upgrade

Most users will never need an Enterprise upgrade if they follow these smart habits. These techniques stretch your token budget and keep conversations running longer.

1️⃣ Start a New Chat When Switching to a New File or Topic
This is the most important habit. If you finish working on one document and want to start on a new uploaded file, start a fresh conversation inside your Project rather than continuing in the same chat. Each conversation gets a fresh 200K token window. Carrying a long conversation forward wastes tokens on old context you no longer need.
💡 Example: Finished reviewing the safety manual? Start a new chat in your Project before uploading the training curriculum.
2️⃣ Keep Project Instructions Concise
Your Project instructions load into every conversation. If they're 2,000 words long, that's 2,000 words of tokens used before you even type your first message. Keep instructions focused and short — 2–4 sentences covering your role, audience, and tone is enough.
3️⃣ Remove Unused Files from Projects
Files uploaded to a Project's knowledge base load into context when relevant. Delete files you're no longer actively using to keep the knowledge base lean and avoid burning tokens on outdated documents.
4️⃣ Be Specific — Avoid Back-and-Forth
Every message and response uses tokens. A well-structured prompt that gets it right on the first try uses far fewer tokens than 5 rounds of vague follow-ups. Use the Role + Context + Task + Format formula from Module 3 to get the right answer faster.
5️⃣ Use Haiku for Simple Tasks
Quick questions, short edits, and simple lookups don't need Opus or Sonnet. Switch to Haiku for lightweight tasks — it's faster and uses your limit more efficiently, saving heavier models for complex work that actually needs them.
6️⃣ Summarize Long Conversations Before They Get Too Long
If you're deep into a long conversation and approaching the context limit, ask Claude to summarize what's been done so far, then start a new chat and paste just the summary as context. This resets your token window while preserving the important information.
💡 Prompt to use: "Summarize everything we've covered in this conversation in 5 bullet points so I can continue in a new chat."
7️⃣ Upload Only What's Needed
Uploading a 300-page document when you only need Claude to review Chapter 4 wastes tokens. Paste or upload only the relevant section of a document when possible, rather than the entire file.

🚨 The Golden Rule for Claude at Quanta Services
🚫Never enter PII, PHI, financial data, customer data, or sensitive company information into Claude. If the information would be confidential in an email, it should not be typed into Claude.
✅Use Claude for general tasks — drafting, brainstorming, writing, explaining — using anonymized or non-sensitive information only.
❓Not sure if it's okay to include? Ask yourself: "Would I be comfortable if this was shared publicly?" If no — don't include it.
